### Summary of the Report

* **Total Cases:** 14
* **Passed:** 9
* **Failed:** 5

> **QA Note:** The "Happy Path" scenarios work perfectly (creating, updating, listing, and deleting products). However, the system fails and returns a `500 Internal Server Error` during negative validation testing (such as negative prices/stocks) and integration tests involving category deletion. The database foreign key (FK) constraint and input validation errors are not properly managed. The development team needs to improve the Global Exception Handling mechanism to catch these database and input errors, converting them into clean, descriptive `400 Bad Request` messages instead of server crashes.
